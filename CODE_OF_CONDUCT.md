# Code Of Conduct

## Our Standard

Participants are expected to be respectful, constructive, and careful with data
rights and privacy.

Examples of positive behavior:

- focusing on technical facts and reproducible reports
- redacting private data, secrets, cookies, and local paths
- respecting source terms, robots rules, and API terms
- giving review feedback that is specific and actionable

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- doxxing or sharing private information
- posting secrets, cookies, session files, browser profiles, or private data
- pressuring contributors to add scraping evasion, proxy/account pools, CAPTCHA
  bypass, paywall bypass, or login-cookie workflows
- knowingly misrepresenting source coverage or data rights

## Enforcement

Maintainers may edit, hide, or remove comments and issues that violate this
code. Repeated or severe violations may result in temporary or permanent
blocking from project spaces.

Sensitive security reports should follow [SECURITY.md](SECURITY.md), not public
issue threads.

Before public launch, maintainers should configure an explicit conduct contact
or moderation path for the GitHub repository.
