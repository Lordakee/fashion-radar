Normalizing there cleanly covers "before rendering and before writing sidecars" without touching SQLite, `edition.json`, the daily report, source acquisition, or collection. No Critical issues.

## Important

**I1 — Cleaning must be applied inside `text_to_local_article_paragraphs`, not just upstream.**
`text_to_local_article_paragraphs` is exported and directly tested (`tests/test_row_one_articles.py:72`). Both call sites (`articles.py:144`, `articles.py:170`) go through it. If cleaning lives only in a separate helper that callers must remember to invoke, the fallback path will regress. Specify that cleaning runs inside this function (or in a single preprocessing step it always calls). The plan should also state whether the existing `test_text_to_local_article_paragraphs_caps_total_text_at_word_boundary` assertion still holds — design cleaning to be idempotent on already-clean plain text so that test does not need weakening.

**I2 — "Split overlong paragraphs" is under-specified and can conflict with the existing budget loop.**
The current loop (`articles.py:82-100`) splits on `\n{2,}` (trafilatura's own paragraph breaks) and enforces a *total* `max_chars` budget with `_truncate_at_word_boundary`. "Split overlong paragraphs into readable paragraphs" needs concrete rules:
- Threshold (chars? sentence count?) that triggers a split.
- Split unit (sentence boundary) and how empty/single-word tails are discarded.
- Ordering vs. truncation: split-then-apply-budget, not truncate-then-split (otherwise you split text that is then thrown away).
Without these, two implementers ship different behavior. Add a test pinning the rule.

**I3 — "Dedupe repeated sentences" can silently drop legitimate content.**
Specify (a) the match key (exact string vs. normalized casefold+punctuation), (b) scope (within one paragraph vs. across the whole article — cross-article is far more aggressive and riskier), and (c) a minimum sentence length so short sentences ("Yes." "No." "Prices rose.") do not collapse. Pin each with a test, including a negative test that legitimate repetition survives.

**I4 — The boilerplate / prefix list must be named and scoped.**
"Original source summary:" is not a feed artifact — it is generated by Fashion Radar itself at `src/fashion_radar/row_one/edition.py:499` and carried into the local-article fallback via `story.summary.en`. Stripping it from rendered local-article paragraphs is desirable, but the plan's framing should be corrected. The initial English-only list ("Read the full story here", "Read more:", image/link markup) will not catch Chinese equivalents ("阅读全文", "点击查看"); either explicitly scope v0.1.x to English boilerplate or make the list data-driven. List the exact initial entries so the set is reviewable and each one gets a test.

**I5 — Exactly one entity-unescape step; specify ordering.**
`html.parser.HTMLParser` with `convert_charrefs=True` (the default) already unescapes entities in `handle_data`. Calling `html.unescape` again afterward double-decodes literal text (e.g. source `&amp;amp;` → parser `&amp;` → unescape `&`, wrong). Pick one: either HTMLParser-based stripping (entities resolved by the parser) **or** regex/stdlib tag strip followed by a single `html.unescape`. Document the chosen order (strip tags → unescape once) and add an idempotence test on plain text containing literal ampersands.

**I6 — Don't introduce a third truncation layer.**
`result.text` is already truncated at `source.article.max_summary_chars` (`src/fashion_radar/collectors/article.py:152`), and `max_summary_chars` is wired to equal `row_one_article.max_chars` via `_source_for_row_one_extraction` (`articles.py:217-225`). The existing `_truncate_at_word_boundary` then re-enforces the budget. The plan's "truncate at word/sentence boundary" must *reuse* `_truncate_at_word_boundary`, not reimplement it. Note also that the extraction path returns already-plain trafilatura text, so HTML stripping there is defensive/idempotent; the artifact concern is primarily on the fallback-summary path. Say so in the plan to keep reviewer mental model accurate.

**I7 — Add a contract guard test for `story.summary`.**
`tests/test_row_one_render.py:1033` and `tests/test_row_one_edition.py:395` assert `"Original source summary:"` remains in `story.summary.en` / meta description. These derive from `story.summary`, not from local-article paragraphs, so stripping the prefix in articles.py must not touch the shared `LocalizedText`. Add a test asserting `story.summary.en` is byte-identical before/after a fallback local-article build, to defend against a future refactor that mutates the shared object.

**I8 — Verify commands.**
Plan says "Verify full pytest and ruff." Per `AGENTS.md`, also run the lockfile check so the public lockfile stays mirror-free: `UV_NO_CONFIG=1 uv lock --check` alongside `uv --no-config run --frozen pytest` and `uv --no-config run --frozen ruff check`. State these explicitly in the plan.
