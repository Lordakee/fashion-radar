# Stage 294 ROW ONE Clean Summary Display Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Clean ROW ONE static HTML summary and metadata display so feed HTML, generated prefixes, and boilerplate do not appear on the website while the underlying story summary and app payload remain unchanged.

**Architecture:** Extract the Stage 293 local-article text cleanup primitives into a narrowly scoped shared ROW ONE text helper, then use that helper for render-time summary display. The helper must sanitize only static HTML presentation paths: lead story summaries, story cards, detail Summary sections, and article meta descriptions. It must not mutate `RowOneStory.summary`, must not change SQLite, and must not change `data/edition.json`.

**Tech Stack:** Python 3.11, ROW ONE static renderer, stdlib `html` / `html.parser` if needed, pytest, ruff, existing `uv --no-config --frozen` verification commands.

---

## Design Constraints

- Keep this stage focused on content display quality, not visual redesign.
- Keep local article extraction and sidecar writing behavior from Stage 293 unchanged.
- Do not change `build_row_one_app_payload`, `data/edition.json`, daily report JSON, or SQLite storage.
- Preserve source attribution, evidence links, and the existing `Open Source Article` action.
- Clean only static HTML display fields derived from `story.summary.en` and `story.summary.zh`.
- Keep `edition.summary` unchanged because it is already generated by ROW ONE and does not carry feed HTML in current fixtures.

## Task 3: Implement Render-Time Summary Cleaning

**Files:**
- Modify: `src/fashion_radar/row_one/templates.py`

- [ ] **Step 1: Add a display-only summary helper**

Implement a helper with this behavior, using the shared helpers from
`row_one.text` rather than duplicating parser/prefix/boilerplate logic:

```python
def _display_summary_text(text: str) -> str:
    # Clean only for static HTML presentation. Do not mutate RowOneStory.summary.
    fallback = normalize_row_one_paragraph(text)
    cleaned = clean_row_one_text(text)
    sentences = clean_row_one_sentences(cleaned, set())
    display_text = normalize_row_one_paragraph(" ".join(sentences))
    return display_text or fallback
```

Required rules:

- Decode HTML entities once, so encoded feed markup becomes parseable.
- Strip HTML tags and ignore `<script>` / `<style>` content.
- Remove only leading generated prefixes:
  - `Original source summary:`
  - `Source summary:`
  - `来源摘要：`
- Remove standalone boilerplate sentences:
  - `Read the full story here.`
  - `Read full story here.`
  - `Read more.`
  - `Read more:`
  - `阅读全文。`
  - `点击查看全文。`
- Normalize whitespace.
- Return the original normalized text if cleaning removes everything.

- [ ] **Step 2: Use the helper in summary display surfaces**

Apply `_display_summary_text` to:

- `render_detail_html` meta description.
- `render_detail_html` Summary section English and Chinese spans.
- `_render_lead_story` English and Chinese summary spans.
- `_render_story_card` English and Chinese summary spans.

Do not apply it to:

- `build_row_one_app_payload`.
- `edition.summary`.
- local article body paragraphs.
