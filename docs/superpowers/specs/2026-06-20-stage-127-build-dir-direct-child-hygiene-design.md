# Stage 127 Build Directory Direct-Child Hygiene Design

## Objective

Make `scripts/check_package_archives.py` enforce that the build output
directory contains only the selected wheel and sdist artifacts plus uv's
generated `.gitignore` marker.

## Problem

The checker help says the build directory should contain exactly one wheel and
one sdist archive. The implementation currently selects one `*.whl` and one
`*.tar.gz`, then validates the contents of those archives. Any unrelated direct
child that does not match either pattern is ignored.

Ignored direct children can include build logs, stale metadata files, hidden
sidecars, or subdirectories. The release path builds into a temporary directory
and runs this checker before upload-readiness decisions, so the checker should
prove the directory is an exact wheel+sdist artifact set.

During implementation, the package smoke exposed a toolchain detail:
`uv --no-config build --out-dir <tmp>` with uv 0.11.7 consistently writes a
direct `.gitignore` file containing `*` beside the wheel and sdist. That marker
is generated by the build tool for the output directory and is not an upload
artifact. The checker should allow that known marker by name while rejecting
every other unexpected direct child.

## Scope

In scope:

- Add tests that reject unexpected direct files and directories beside the
  selected wheel and sdist.
- Add a test that accepts uv's generated `.gitignore` marker beside the selected
  wheel and sdist.
- Add a deterministic helper in `scripts/check_package_archives.py` that scans
  direct children of the build directory and reports every unexpected child
  except the allowed `.gitignore` marker.
- Keep duplicate wheel/sdist detection behavior unchanged.
- Keep archive-internal validation behavior unchanged.

Out of scope:

- No package metadata derivation or `pyproject.toml` refactor.
- No release docs or CI workflow changes.
- No dependency changes.
- No `uv.lock` changes.
- No connector, scraping, browser automation, platform API, monitoring,
  scheduling, source acquisition, demand proof, ranking, coverage verification,
  or compliance/audit product behavior.

## Architecture

This is a release-checker guardrail:

1. `validate_build_dir()` keeps selecting the single wheel and single sdist
   first, preserving existing missing/duplicate archive messages.
2. After successful selection, a new direct-child helper compares
   `build_dir.iterdir()` against the selected wheel and sdist paths plus the
   allowed `.gitignore` marker name.
3. The helper returns sorted, deterministic errors such as
   `build directory contains unexpected direct child: build.log`.
4. `validate_build_dir()` returns direct-child errors before archive-internal
   errors so a contaminated build directory fails clearly.

The helper is direct-child-only. It does not recurse, resolve symlinks, or
inspect sidecar contents.

## Expected Behavior

After implementation:

- A build directory with exactly one valid wheel and one valid sdist still
  passes.
- A build directory with exactly one valid wheel, one valid sdist, and uv's
  direct `.gitignore` marker still passes.
- A build directory with a valid wheel, valid sdist, and `build.log` fails with
  `build directory contains unexpected direct child: build.log`.
- A build directory with a valid wheel, valid sdist, and a `metadata/`
  directory fails with
  `build directory contains unexpected direct child: metadata`.
- A build directory with multiple unexpected children reports all of them in
  deterministic name order.
- A build directory with uv's `.gitignore` marker plus another unexpected
  direct child still fails for the unexpected child.
- Existing missing and duplicate archive errors remain unchanged.

## Risks

- A dirty local `dist/` containing checksums, signatures, attestations, logs, or
  old artifacts will fail. This is intentional for the current checker because
  its documented contract is an exact wheel+sdist archive set, with only uv's
  generated `.gitignore` marker tolerated.
- If a future release process signs packages before running this checker, the
  process should either run this checker earlier or add an explicit allowlist in
  a separate design.
- Path comparisons should avoid `resolve()` so the helper does not follow
  symlinks or depend on filesystem resolution behavior.
- The `.gitignore` allowance is intentionally by name. uv 0.11.7 currently
  writes a one-byte `*` marker, and future uv marker changes should be handled
  in a separate checker update.

## Verification

Focused verification:

```bash
uv --no-config run --frozen pytest tests/test_package_archives.py -k "unexpected_direct or build_directory or gitignore" -q
uv --no-config run --frozen pytest tests/test_package_archives.py -q
uv --no-config run --frozen ruff check scripts/check_package_archives.py tests/test_package_archives.py
uv --no-config run --frozen ruff format --check scripts/check_package_archives.py tests/test_package_archives.py
tmp_build="$(mktemp -d)"
uv --no-config build --out-dir "$tmp_build"
uv --no-config run --frozen python scripts/check_package_archives.py "$tmp_build"
```
