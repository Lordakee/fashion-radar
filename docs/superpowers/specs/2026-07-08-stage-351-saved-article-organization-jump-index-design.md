# Stage 351 ROW ONE Saved Article Organization Jump Index Design

## Objective

Stage 351 adds a generated-site-only Saved Article Organization Jump Index
inside `articles/index.html`. The index gives readers a compact set of same-page
shortcuts into the existing saved article organization surfaces, so they can
move quickly between local article content groups, source routes, signal facets,
and the daily signal leaderboard without scanning the full page.

## Product Gap

Stages 346-350 added several useful local saved article surfaces, but the
article library page now has many independent sections. Readers need a small
orientation block that answers: where should I jump first if I want to read by
content group, by source, or by signal?

The Jump Index closes that gap as navigation only. It is not a new summary,
ranking, analytics signal, recommendation layer, market heat indicator, or
external source monitor.

## Architecture

This is a generated-site-only presentation change that reuses existing in-memory
objects already built for the saved article library page:

- `src/fashion_radar/row_one/saved_article_organization_jump_index.py` builds a
  compact view model from existing safe saved article organization structures.
- The builder accepts existing optional surfaces such as
  `RowOneSavedArticleContentOrganization`,
  existing saved article source route view models,
  `RowOneSavedArticleSignalFacets`, and
  `RowOneSavedArticleDailySignalLeaderboard`.
- The builder creates up to three navigation groups:
  `Content`, `Sources`, and `Signals`.
- Each group contains capped items with localized labels, support counts, and
  same-page href anchors that already exist in `articles/index.html`.
- `src/fashion_radar/row_one/render.py` wires the view model into
  `_write_saved_article_library_page(...)` only.
- `src/fashion_radar/row_one/templates.py` renders the section after the saved
  article daily summary and before the local article content organization
  section, so it acts as an orientation index for the generated saved article
  library page.

The feature creates no schema, SQLite table, JSON artifact, route family,
collector, extraction path, ranking model, LLM call, scheduler, deployment,
app-facing contract field, or compliance-review product behavior.

## Rendering Behavior

When at least one safe navigation item exists, Stage 351 renders a
`saved-article-organization-jump-index` section in `articles/index.html`.

Each navigation group should show:

- a localized group heading;
- up to four navigation items;
- a localized label;
- a factual count copied from an existing local surface;
- an existing same-page href such as
  `#saved-article-content-organization`,
  the first rendered per-source route anchor such as
  `#saved-article-source-vogue-business`,
  `#saved-article-signal-facets`, or
  `#saved-article-daily-signal-leaderboard`.

The section is omitted when no existing local surface can produce a safe item.
It should not appear on the homepage or detail article pages.

## Data Sources

Stage 351 should reuse only these already-built local saved article structures:

- `RowOneSavedArticleContentOrganization.groups`
- existing saved article source route sections already rendered in
  `articles/index.html`, using their concrete per-source anchor ids
- `RowOneSavedArticleSignalFacets`
- `RowOneSavedArticleDailySignalLeaderboard`

It should not inspect raw article text, raw references, source URLs, imported
signals, external search results, social platform data, or downloaded article
HTML.

## Safety And Caps

All visible labels and counts must be escaped by the template layer. All hrefs
must be same-page fragment links to anchors already rendered by the saved
article library template.

The implementation should not invent an aggregate source-route anchor. Source
items must point to existing per-source anchors generated by the saved article
source route logic, such as `#saved-article-source-vogue-business`, and render
tests must verify every Jump Index href has a matching `id` in the same
`articles/index.html` output.

Caps:

- up to three groups;
- up to four items per group;
- no outbound links;
- no full article body text;
- no copied source brief bullets;
- no copied article summaries.

## Out Of Scope

Stage 351 does not add article downloading, social-platform collection,
external tool adapters, scraping, browser automation, platform APIs,
account/session behavior, extraction changes, LLM summaries, ranking,
recommendation, analytics, personalization, scheduling, deployment, app
contract changes, schema changes, new route families, or compliance-review
product behavior.

It also does not create
`data/saved-article-organization-jump-index.json`,
`data/local-article-organization.json`,
`data/article-organization.json`,
`saved-article-organization-jump-index.html`,
`local-article-organization.html`, or any new JSON/HTML artifact.

## Acceptance Criteria

- `articles/index.html` renders a `saved-article-organization-jump-index`
  section after the saved article daily summary and before local content
  organization when at least one safe item exists.
- The section links only to same-page anchors already rendered in the saved
  article library page.
- Source navigation uses concrete existing per-source anchors rather than a
  non-existent aggregate `#saved-article-source-routes` target.
- Render tests verify every Jump Index href has a matching same-page `id`.
- Content, source, and signal groups are capped and preserve existing page
  order instead of support-count sorting.
- Empty input surfaces omit empty groups and omit the whole section when all
  groups are empty.
- Labels and counts are escaped by the template layer.
- The feature does not appear on the homepage.
- No app-facing contract, schema, JSON artifact, route family, fetching,
  extraction, ranking, LLM, scheduling, deployment, analytics, personalization,
  recommendation, or compliance-review product behavior changes.
- Documentation and workflow guards state the generated-site-only boundary for
  Stage 351.
