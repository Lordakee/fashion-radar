# Stage 355 Saved Article Local Section Binder Design

## Goal

Improve each generated local article page (`articles/<story-id>.html`) with a
compact scan-first section binder that indexes the saved article's existing
content sections, cited references, and saved paragraph anchors before readers
enter the full local article body.

The feature should make one saved article easier to file, skim, and inspect
locally. It should not act as another read-next module, outbound link hub, or a
duplicate of the existing Local Article Information cards.

## Product Scope

- Add an optional `Saved Article Local Section Binder` section to generated
  local article pages.
- Render it after the Stage 354 local reading companion and before the existing
  `#local-article` body.
- Show a compact binder/index, not full section cards:
  - the current article title and source;
  - ordered content-section rows from `RowOneLocalArticle.content_sections`,
    preserving each original content-section position for anchors;
  - item chips from existing content-section item labels;
  - capped reference labels already attached to those items;
  - cited saved paragraph counts;
  - same-page jumps to `#local-article-content-section-N`;
  - same-page jumps to valid cited `#local-article-paragraph-N` anchors;
  - an optional unfiled paragraph row for saved paragraphs not cited by any
    section.

## Non-Goals

- No app contract, schema, runtime, manifest, or JSON artifact changes.
- No new article sidecars, generated data files, route families, or local
  storage behavior.
- No new scraping, source fetching, extraction, scoring, ranking, LLM calls,
  social connectors, scheduling, deployment, analytics, personalization, or
  compliance-review behavior.
- No mutation of stored article text, paragraph indices, content sections,
  references, or extracted source records.
- No outbound links or external article fetches.

## Data Sources

Reuse existing in-memory render inputs:

- `RowOneStory`
- `RowOneLocalArticle`
- `RowOneLocalArticle.content_sections`
- `RowOneLocalArticle.content_sections[*].items`
- `RowOneLocalArticle.content_sections[*].items[*].references`
- `RowOneLocalArticle.content_sections[*].items[*].paragraph_indices`
- `RowOneLocalArticle.paragraphs`
- `RowOneLocalArticle.paragraphs_zh`

The builder must sanitize route and identity inputs with existing helpers:

- `safe_local_article_story_id`
- the article/story id equality check already used by local page features

All links generated by this feature are renderer-owned same-page fragments.

## Rendering Rules

- Return `None` when `story.id != local_article.story_id`, when the story id is
  unsafe, or when no meaningful section or unfiled paragraph rows can be built.
- Preserve `content_sections` order and item order.
- Use original `content_sections` positions for
  `#local-article-content-section-N`, even when earlier sections are skipped or
  section rows are capped.
- Treat `content_sections[*].items[*].paragraph_indices` as zero-based source
  paragraph indices; render anchors as one-based `#local-article-paragraph-N`.
- Skip boolean, negative, duplicate, out-of-range, and blank paragraph indices.
- Use English paragraph excerpts from `paragraphs` and aligned Chinese excerpts
  from `paragraphs_zh` when present.
- Deduplicate references while preserving first-seen order.
- Cap sections, item chips, references, paragraph links, and excerpts so the
  article page remains compact.
- Escape all rendered text in `templates.py`.

## Acceptance Criteria

- Local article pages can render the binder without changing site contracts or
  generated data artifacts.
- The binder appears only on eligible `articles/<story-id>.html` pages.
- The binder appears after `.saved-article-local-reading-companion` and before
  `id="local-article"`.
- All binder links are same-page `#local-article-*` fragment links generated
  from validated positions.
- Tests cover builder grouping, invalid paragraph indices, unfiled paragraphs,
  template rendering, generated-site integration, docs references, and workflow
  guard coverage.
